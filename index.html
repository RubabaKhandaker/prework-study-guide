HTML

  • "HTML Attributes" define additional information about "HTML Elements"
  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • Semantic HTML elements make HTML easier to read, and makes sure a webpage is accessible to a wide variety of users e.g. those who use assistive technology
  • HTML elements are case sensitive
  • An element nestled inside of another element is called a "child element"

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • "Inline CSS" is used directly inside an HTML file, and it dictates the style of a singular element
  • "Internal Style Sheet" is used to dictate the style of all tagged elements in a webpage
  • "External Style Sheet" is the most common, and is where an external CSS file is created and linked into HTML code

Git

  • git status: checks out what branch we are currently on
  • git checkout -b branch name: creates a new branch and switches to it
  • git add: adds modifications in the current working branch to the staging area
  • add -A: indicates to commit all files to a working branch
  • -m: associates a message with our commit

JavaScript

  • A variable is a named container that allows us to store data in our code
  • String: A sequence of text enclosed in single or double quote marks
  • Boolean: A True/False value
  • Array: A structure that allows multiple values to be stored in a single reference
  • Object: Everything in JS is an object, and can be stored in a variable
  • Control flow: The order in which a computer executes code from a script
